## 実装スケジュール

### Phase 1（最初の1ヶ月）
- 実用的アドバイス系: 4記事
- 体験談系: 2記事
- データ分析系: 1記事
- リスト系: 2記事
**合計: 9記事**

### Phase 2（2-3ヶ月目）
- 各カテゴリーから均等に選択
- 月10-12記事のペースで追加
- ユーザーフィードバックに基づいて調整

### Phase 3（4ヶ月目以降）
- パフォーマンスの高い記事タイプに注力
- 既存記事の更新と最適化
- ユーザー投稿・ゲスト記事の検討

## SEO最適化のポイント

1. **キーワード戦略**
   - 各記事に主要キーワード1つ + 関連キーワード3-5個
   - タイトル、メタディスクリプション、H2に自然に含める
   - ロングテールキーワードを狙う

2. **内部リンク**
   - 各記事から3-5個の関連ページへリンク
   - タイプ詳細、職業、相性ページへの導線
   - 新しい記事から古い記事へもリンク

3. **構造化データ**
   - ArticleStructuredData を各記事に追加
   - 著者、公開日、更新日を明記
   - パンくずリストの実装

4. **画像・メディア**
   - 各記事にアイキャッチ画像
   - インフォグラフィックの活用
   - Alt テキストの最適化

5. **読みやすさ**
   - 見出しの階層構造
   - 短い段落（2-3文）
   - 箇条書きの活用
   - 結論を先に書く

## メトリクス測定

以下の指標を月次で追跡：
- ページビュー数
- 平均滞在時間
- 直帰率
- 検索順位（主要キーワード）
- ソーシャルシェア数
- コンバージョン率（診断ツールへの誘導）

## 記事作成ガイドライン

### 基本構造
1. **導入**（100-150文字）
   - 問題提起または共感を呼ぶエピソード
   - 記事で得られる価値の明示

2. **本文**（1500-2500文字）
   - H2, H3 で明確に構造化
   - 具体例・エピソードを豊富に
   - データや引用で信頼性を高める

3. **結論**（100-150文字）
   - 要点のまとめ
   - 行動を促すCTA
   - 関連ページへのリンク

### 文体
- 「です・ます」調で統一
- 親しみやすく、専門的すぎない
- 読者に語りかける口調
- ポジティブなトーン

### 必須要素
- メタディスクリプション（120-160文字）
- アイキャッチ画像
- 関連記事リンク（最低3つ）
- CTA（診断、他のページへの誘導）

## 記事テンプレートファイルの場所

新しい記事を作成する際は、`src/content/blog/` ディレクトリに以下の形式でMarkdownファイルを作成してください：

```markdown
---
title: "記事タイトル"
excerpt: "記事の要約（120-160文字）"
author: "著者名"
publishDate: "2025-01-06"
image: "/images/blog/記事画像.jpg"
tags: ["MBTI", "INFP", "キャリア"] # 適切なタグを選択
---

記事本文をここに書く...
```

## まとめ

この計画を実行することで：
- **検索トラフィック**: 月間50%以上の増加を目指す
- **滞在時間**: 平均3分以上に延長
- **ページ数**: 6ヶ月で50-60記事追加
- **権威性**: 包括的なMBTIリソースとしての地位確立

定期的にパフォーマンスを分析し、効果の高いコンテンツタイプに注力することで、持続的な成長を実現します。
